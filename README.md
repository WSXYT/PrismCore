# PrismCore

Windows 系统优化工具，基于 WinUI 3 构建，提供内存管理、CPU 智能调度、延迟诊断、磁盘清理、网络修复和启动项管理六大核心功能。

## 功能概览

### 仪表盘
实时系统健康评分，综合 CPU、内存、磁盘、DPC/ISR 延迟等指标，一键智能优化。

### 内存优化
- **备用列表清理** — 释放系统缓存占用的物理内存
- **工作集修剪** — 收缩后台进程的内存占用
- **空闲进程分页** — 将长时间不活跃的进程内存分页到磁盘
- **虚拟内存管理** — 智能检测分页文件状态，提供扩展建议

### CPU 智能调度（ProBalance）
- 基于 EWMA 趋势预测 + Z-score 异常检测算法
- 自动降低后台高 CPU 进程的优先级
- 混合架构 CPU（大小核）的亲和性自动调整
- 白名单保护系统关键进程和音频进程

### DPC/ISR 延迟诊断
- PDH 系统计数器实时监控 DPC 时间、中断时间、队列深度
- ETW 内核级事件追踪，精确定位问题驱动
- 内置常见驱动数据库（NVIDIA、AMD、Realtek 等），提供修复建议

### 深度磁盘清理
- **快速扫描** — 临时文件、Electron 应用缓存
- **深度扫描** — MFT 快速大文件扫描（500MB+）、系统组件清理、旧驱动清理
- WinSxS 组件存储压缩
- 注册表孤立项扫描
- Windows 更新缓存清理

### 网络修复工具箱
- DNS 缓存刷新
- Winsock 目录重置
- TCP/IP 协议栈重置

### 启动项管理
- 注册表 Run 键值枚举
- 任务计划程序枚举
- 一键启用/禁用启动项

## 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 1903 (Build 17763) 及以上 |
| 运行时 | .NET 10（应用自包含，无需单独安装） |
| 权限 | **管理员权限**（必需） |
| 架构 | x64 / x86 / ARM64 |

## 安装与运行

### 下载安装（推荐）

1. 前往 [GitHub Releases](https://github.com/WSXYT/PrismCore/releases) 下载对应架构的安装包：
   - `PrismCore-*-win-x64-Setup.exe` — 64 位 Intel/AMD（大多数电脑）
   - `PrismCore-*-win-arm64-Setup.exe` — ARM64（Surface Pro X、骁龙笔记本等）
   - `PrismCore-*-win-x86-Setup.exe` — 32 位系统
2. 双击运行 Setup.exe，系统弹出 UAC 提权提示后自动完成安装
3. 安装后桌面和开始菜单会自动创建快捷方式

> 应用支持自动更新，安装后可在「更新」页面配置更新策略。

### 卸载

Windows 设置 → 应用 → 已安装的应用 → 找到 PrismCore → 卸载

### 从源码构建

**前置条件：**
- Visual Studio 2022 17.12+ 或 .NET 10 SDK
- Windows App SDK 工作负载

```bash
# 克隆仓库
git clone <仓库地址>
cd PrismCore

# 构建
dotnet build PrismCore/PrismCore.csproj -c Release

# 发布 MSIX（以 x64 为例）
dotnet publish PrismCore/PrismCore.csproj -c Release -r win-x64
```

## 使用说明

### 首次启动

1. 启动后进入**仪表盘**页面，显示系统健康评分和实时指标
2. 点击「智能优化」按钮执行一键优化（清理备用列表 + 修剪工作集）
3. 应用会在后台持续监控系统状态，超过阈值时自动优化

### 页面导航

应用左侧为导航栏，包含五个页面：

- **仪表盘** — 系统概览和一键优化
- **清理** — 磁盘空间清理（快速/深度模式）
- **优化** — 内存优化、启动项和进程管理
- **工具箱** — 网络诊断与修复工具
- **更新** — 检查更新、一键升级、自动更新设置
- **设置** — 所有可配置参数

### 磁盘清理流程

1. 进入「清理」页面
2. 选择扫描模式（快速 / 深度）
3. 点击「扫描」，等待扫描完成
4. 在结果列表中勾选要清理的项目
5. 点击「清理」执行删除

### 网络修复

进入「工具箱」页面，根据网络问题选择对应操作：

- **DNS 异常** → 刷新 DNS 缓存
- **网络连接异常** → 重置 Winsock
- **协议栈损坏** → 重置 TCP/IP（需重启）

### 系统托盘

关闭窗口后应用最小化到系统托盘，右键托盘图标可：
- 显示主窗口
- 退出应用

## 设置说明

所有设置实时保存，重启后自动恢复。设置文件位于：

```
%LocalAppData%\PrismCore\settings.json
```

### 后台自动优化

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用自动优化 | 开启 | 内存超过阈值时自动清理 |
| 内存阈值 | 60% | 触发自动优化的内存使用率（30-95%） |
| 自动优化间隔 | 10 秒 | 后台检测周期（5-120 秒） |

### 虚拟内存

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 自动分页文件 | 开启 | 内存不足时自动创建临时分页文件 |
| 分页扩展阈值 | 70% | 提交比超过此值时扩展分页文件（50-90%） |
| 智能建议 | 开启 | 检测到分页文件需求时弹出建议 |

### CPU 智能调度

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用智能调度 | 开启 | 自动约束后台高 CPU 进程 |
| 系统 CPU 阈值 | 45% | 系统总 CPU 超过此值时触发调度（30-95%） |
| 进程 CPU 阈值 | 8% | 单进程 CPU 超过此值时被约束（5-50%） |

### 异常检测

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 启用异常检测 | 开启 | 基于 EWMA + Z-score 检测 CPU 异常尖峰 |
| Z-score 阈值 | 3.0 | 越小越敏感，推荐 2.0-5.0 |
| EWMA Alpha | 0.3 | 平滑系数，越大对近期数据越敏感（0.05-0.95） |

### 内存优化策略

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 清理备用列表 | 开启 | 释放系统缓存占用的内存 |
| 修剪工作集 | 开启 | 修剪后台进程的内存工作集 |
| 智能分页空闲进程 | 开启 | 将长时间空闲的进程分页到磁盘 |

### 系统监控

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| DPC/ISR 延迟监控 | 开启 | 监控内核延迟，检测驱动问题 |

### 恢复默认

在设置页面底部点击「恢复默认设置」可将所有配置重置为出厂值。

## 技术架构

```
PrismCore/
├── Models/          # 业务逻辑层（内存管理、CPU 调度、清理引擎等）
├── ViewModels/      # 视图模型层（数据适配、命令绑定）
├── Views/           # UI 层（WinUI 3 XAML 页面）
├── Helpers/         # 工具类（常量、P/Invoke、权限管理、托盘图标）
├── Converters/      # XAML 值转换器
└── Assets/          # 应用图标和资源
```

- **架构模式：** MVVM（Model-View-ViewModel）
- **UI 框架：** WinUI 3（Windows App SDK）
- **MVVM 框架：** CommunityToolkit.Mvvm
- **日志：** Serilog（文件 + 调试输出）
- **错误追踪：** Sentry
- **打包分发：** Velopack（自动更新 + 安装/卸载管理）

## 日志

运行日志保存在：

```
%LocalAppData%\PrismCore\logs\prismcore-20260224.log
```

按日期滚动，可用于排查问题。

## 注意事项

- 应用需要**管理员权限**运行，否则内存清理、驱动清理等功能无法正常工作
- 深度清理中的「旧驱动清理」和「WinSxS 压缩」操作不可逆，建议先创建系统还原点
- 注册表清理前会自动备份到 `%LocalAppData%\PrismCore\RegBackup\`
- 重置 TCP/IP 协议栈后需要重启计算机
- 应用为单实例运行，不支持同时打开多个窗口
